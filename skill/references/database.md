## mameyugo/jsonq — JSON File Persistence

### What it is
PHP extension written in Rust providing `.json` file storage with MongoDB-style
queries, indexing, schema validation, and ACID transactions.
Requires the native `jsonq.so` extension loaded in PHP.

### Requirements
- PHP >= 8.1
- `ext-jsonq` installed (`php -m | grep jsonq`)
- Data folder with PHP write permissions

### Mandatory Project Rules

- [ ] **Never edit the `indexes/` folder** — self-managed by JsonQ
- [ ] **IDs**: [describe detected ID policy — UUID, manual auto-increment, etc.]

### Performance Notes
- SIMD parsing (simd-json) + Arc cache with mtime invalidation
- Hash indexes allow O(1) lookups on indexed fields
- For large collections (>10k records), verify indexes are defined
- File locking can bottleneck under high concurrency
